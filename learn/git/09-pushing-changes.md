# Senaryo 9: Pushing Changes
## Öğrenilenler

-git push Yerel commit'leri uzak depoya yükleyerek çalışmayı başkalarıyla paylaşmayı sağlar.
-u (upstream) Parametresi Yerel dalı uzak daldaki karşılığına bağlar. Bu sayede sonraki işlemlerde sadece \`git -push\` yazmak yeterli olur.
-Yeni Dalı Paylaşma Yerel bilgisayarda oluşturulan bir dalı uzak repoya aktarmak için \`git push -u origin - - -<dal-adi>\` komutu kullanılır.
-Öğrenilen İşbirliği yapmanın temel taşı, yereldeki kodun uzak depoya "push" edilmesidir.
